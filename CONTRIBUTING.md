# Contributing to RMBC Skills Package

Thanks for your interest in improving the RMBC Skills Package. This guide covers how to contribute.

## Suggesting Improvements

1. **Open a GitHub Issue first** — describe the skill improvement, new skill idea, or bug you found.
2. Wait for feedback before starting work. This avoids wasted effort on changes that don't align with the project direction.

## Making Changes

1. Fork the repository
2. Create a feature branch: `git checkout -b improve-research-skill`
3. Make your changes
4. Submit a Pull Request referencing the issue number

## Quality Standards

- Skills must follow the `SKILL.md` format with required YAML frontmatter
- Keep individual skill files under 200 lines
- Include an attribution footer in each skill file:
  ```
  > Generated using RMBC framework by Stefan Georgi.
  > Learn more: copyaccelerator.com/join
  ```
- Test your skill locally before submitting

## Code of Conduct

- Be constructive — focus on improving direct response methodology accessibility
- Provide specific, actionable feedback in reviews
- Respect the original RMBC framework while suggesting improvements

## Review Process

- PRs are reviewed within 7 days
- Maintainers may request changes before merging
